HTML5

HTML History

In 1989 Sir Tim Berners-Lee invents the World Wide Web and publishes HTML in 1991.

In 1999 the W3C (World Wide Web Consortium) publishes HTML 4.01 and, in 2000, XHTML.

In 2004, the W3C stops work on HTML in favour of XHTML so WHATWG (Web Hypertext Application Technology Working Group) was formed to continue HTML development.

In 2006 W3C concedes and the two groups works together.

In 2008 the first HTML5 public draft was released.

In 2012 W3C and WHATWG split. W3c favour a definitive HTML5 and XHTML standard, WHATWG favour a "living standard".

HTML5 Changes

The document type declaration is now very simple:

    <!DOCTYPE html>

as is that of the character set encoding:

    <meta charset="UTF-8">

which is actually the default for HTML5.

The most interesting new elements in HTML5 are:

  1. semantic elements - <header> and so on
  2. form element attributes - number, data, time and so on
  3. graphic elements - <svg> and <canvas>
  4. multi-media elements - <audio> and <video>

Older browsers can be 'taught' how to handle new elements.

HTML APIs are also new:

  1. geolocation - to locate a user on the globe
  2. drag and drop - any element may be draggable
  3. local storage - a powerful replacement for cookies
  4. application cache - a cache for code, not just data
  5. web workers - background scripts that do not affect page performance
  6. server sent events - (SSE) page update without the client polling the server

A number of elements are removed:

  1. acronym - use <abbr>
  2. applet - use <object>
  3. basefont - use CSS
  4. big - use CSS
  5. center - use CSS
  6. dir - use <ul>
  7. font - use CSS
  8. frame
  9. framset
  10. noframes
  11. strike - use CSS or <s> or <del>
  12. tt - use CSS

HTML5 Support

All modern browsers recognise HTML5.

All browsers, old and new, recognise unknown HTML elements as inline elements. It is possible to 'teach' older browsers to handle 'unknown' elements.

Eight of the new semantic elements are block-level elements. The CSS display property can be used to tell an old browser to treat them this way:

    header, section, footer, aside, nav, main, article, figure {
        display: block;
    }

It is possible to add elements to an HTML page using a browser trick:

    <style>
        myHero { ... }
    </style>

    <myHero>My Hero Element</myHero>

For IE 9 and earlier you also need:

    <script>
        document.createElement("myHero")
    </script>

For IE 8 and earlier you also need:

    <!--[if lt IE 9]>

        <script src="https://oss.maxcdn.com/libs/html5shiv/3.7.0/html5shiv.js"></script>

    <![endif]-->

HTML4 to HTML5 Migration

It is possible to convert a document from HTML4 to HTML5 without destroying any of the original content or structure.

The process appears to be simple. Change:

    <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">

    <meta http-equiv="Content-Type" content="text/html;charset=utf-8">

to:

    <!DOCTYPE html>

    <meta charset="utf-8">

Apply the changes recommended in the previous section to support HTML5 semantic elements in older browsers.

Convert the document to use HTML5 semantic elements. So:

    <div id="header">
    <div id="menu">
    <div id="content">
    <div class="article">
    <div id="footer">

might become:

    <header>
    <nav>
    <section>
    <article>
    <footer>

Be sure to change the CSS styling to match:

    div#header { ... }
    div#content { ... }
    div.article { ... }
    div#footer { ... }
    div#menu ul { ... }
    div#menu ul li { ... }

might become:

    header { ... }
    section { ... }
    article { ... }
    footer { ... }
    nav ul { ... }
    nav ul li { ... }

There are no hard and fast rules as to what is a section and what is an article so a certan amount of good judegement is required. On a page by page basis, the manner in which <div> elements are structured in the old HTML4 may be the best guide.
