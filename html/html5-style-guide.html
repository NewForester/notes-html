HTML5 Style Guide

This section is coding conventions and a style guide for HTML5. Much of it has already been recommended.

HTML5 is not as rigorous as XHTML and although XHTML may have fallen out of favour, it is recommended the discipline of XHTML be retained.

At the document level:

  1. always declare the correct document type;
  2. do not leave out the html element: use it to specify the document language;
  3. do not leave out the body or the head elements;
  4. the title element is required: make it meaningful;
  5. define the language and character set as early as possible;
  6. always set the view port;
    <!DOCTYPE html>
    <html lang="en-GB">
    <head>
      <meta charset="UTF-8">
      <meta name="viewport" content="width=device-width, initial-scale=1.0">
      <title>The Answer to Life, the Universe and Everything</title>
    </head>
    <body>
      <!-- Whatever you want.  Whatever you need. -->
    </body>
    </html>

With respect to white space and comments:

  1. avoid long code lines: 80 characters is suggested as a maximum;
  2. do not add blank lines without a reason: use for readability between large or logical code blocks;
  3. use 2 spaces for indentation: do not use the tab key;
  4. it is not necessary to indent every element;
  5. short comments should be written on one line;
  6. longer comments should be opened and closed on separate lines;
    <!-- This is a short comment -->

    <!--
      A longer comment.
      Spread over several lines.
    -->

For elements:

  1. write element names in lower case: certainly avoid mixed case;
  2. close all elements: never leave off the closing </element>;
  3. close empty elements: use />, not just >;
  4. write attribute names in lower case: certainly avoid mixed case;
  5. quote all attribute values: they are required when the value contains spaces;
  6. always supply an alt attribute to image elements;
  7. also define the image width and height to reduce flicker;
  8. do not use spaces around the = between attribute names and their values;

For styles and style sheets:

  1. use simple syntax for linking to style sheets: the type attribute is not necessary;
  2. short CSS rules may be written on one line;
  3. longer CSS rules should be split over a number of lines;
    <link rel="stylesheet" href="styles.css">

    body { border-style: double; }

    body {
      background-color: lightgrey;
      font-family: "Arial Black", Helvetica, sans-serif;
      font-size: 16em;
      color: black;
    }

Furthermore:

  1. place the opening bracket on the same line as the selector;
  2. use one space before the opening bracket;
  3. use 2 spaces for indentation;
  4. use semicolon after each property-value pair, including the last;
  5. only use quotes around values if the value contains spaces;
  6. place the closing bracket on a new line, without leading spaces;
  7. avoid lines over 80 characters;

For scripts:

  1. use simple syntax for loading external scripts: the type attribute is not necessary;

Javascript is case sensitive. The following are not equivalent:

    var obj = getElementById("Exemplar");

    var obj = getElementById("exemplar");

For file names:

  1. use lower case file names: these work with case and case insensitive web servers and file systems;
  2. use the file extensions .html, .css and .js;
  3. assume the server will default the page name to "index.html";
