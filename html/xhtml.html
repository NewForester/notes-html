HTML and XHTML

XHTML is HTML written as XML:

  1. XHTML stands for eXtensible HyperText Markup Language;
  2. it is HTML defined as an XML application
  3. it is supported by all major browsers
  4. it is almost identical;
  5. it is stricter;

By writing XHTML documents, you adhere to a stricter standard, which should mean better documents. There is certainly nothing to be lost by doing so.

Differences in document structure:

  1. DOCTYPE is mandatory
  2. the xmlns attribute of the <html> element is mandatory
  3. <html>, <head>, <title> and <body> elements are mandatory
  4. there can only be one root element

For example:

    <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN"  "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">

    <html xmlns="http://www.w3.org/1999/xhtml">

    <head>
      <title>Title of document</title>
    </head>

    <body>
      some content
    </body>

    </html>

XHTML elements:

  1. must be closed
  2. must be properly nested
  3. must be in lower case

Elements must be closed so <br /> not <br>.

XHTML attribute:

  1. names must be in lower case
  2. values must be quoted
  3. values cannot be minimised

Minimisation is not allowed, so checked="checked" not simply checked.

It is possible to validate XHTML documents on-line.
